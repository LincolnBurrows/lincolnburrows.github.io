计算机漏洞安全相关概念


1. POC
POC,Proof of Concept,中文意思“观点证明”，这个短语在漏洞报告中使用，漏洞报告中的POC则是一段说明或者一个攻击的样例，使得读者能够确认这个漏洞的真实存在

2. EXP
EXP, Exploit，中文意思是“漏洞利用”，意思是一段对漏洞如何利用的详细说明或者一个演示的漏洞攻击代码，可以使得读者完全了解漏洞的机理以及利用方法

3. VUL
VUL， Vulnerability的缩写，泛指漏洞

4. CVE漏洞编号
CVE的英文全称是“Common Vulnerabilities&Exposures”公共漏洞和暴露，例如CVE-2015-0057、CVE-1999-0001等等，CVE就好像是一个字典表，为广泛认同的信息安全漏洞或者已经暴露出来的弱点给出一个公共的名称，如果在一个漏洞报告中指明一个漏洞，如果有CVE名称，你就可以快速地在任何其他CVE兼容的数据库中找到相应修补信息，解决安全问题。

网站https://cve.mitre.org 或者 中文社区http://www.scap.org.cn 根据漏洞CVE编号搜索该漏洞的介绍

5. 0DAY漏洞和0DAY攻击
在计算机领域中0Day（zero-dayexploit）通常是指还没有补丁的安全漏洞，而0Day攻击则是指利用这种漏洞进行的攻击，提供该漏洞细节或者利用程序的人通常是该漏洞的发现者，0Day的利用对于网络安全具有巨大的威胁，0Day不但是黑客最爱，而且掌握多少0Day也是评价黑客技术水平的一个重要参数。
